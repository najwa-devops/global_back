# V1 Fix Notes - OCR Parsing Reliability

## Scope
This document summarizes the latest fix set applied after observing cases where OCR text was complete but parsed fields were missing or mis-mapped.

## Reported Symptom
For some invoices, parser output was incomplete/misaligned:
- `invoiceDate` empty
- `amountHT` empty
- `tva` got `amountHT` value
- `amountTTC` got `tva` value

Even when OCR raw text contained all totals and date information.

## Root Causes Identified
1. Date regex was too strict for OCR-noisy formats.
   - Example OCR: `Date facturation'. 08101 12025`
2. Totals extraction relied mostly on inline regex and was weak for split blocks:
   - `TotalHT`
   - `Total TVA 20%`
   - amounts on following lines (`7 000,00`, `1 400,00`)
3. DB fallback cache could remain empty for a field and not recover.
4. `data.sql` had a broken `amountHT` insert block and minor regex typos, reducing available fallback patterns.

## Changes Implemented

### 1) Date extraction/normalization hardened
**File:** `src/main/java/com/invoice_reader/invoice_reader/servises/dynamic/DynamicFieldExtractorService.java`

- Expanded `invoiceDate` patterns to accept OCR-corrupted separators/chars.
- Added date normalization method:
  - converts common OCR chars (`O/o -> 0`, `I/l -> 1`)
  - supports loose forms and reconstructs valid `DD/MM/YYYY`
  - handles noisy `08101 12025`-style input

### 2) Totals block parsing added (label -> next amount line)
**File:** `src/main/java/com/invoice_reader/invoice_reader/servises/dynamic/DynamicFieldExtractorService.java`

- Added pre-pass over totals zone:
  - detect labels (`Total HT`, `Total TVA`, `Total TTC`)
  - capture inline amount OR next-line amount
  - map to canonical fields: `amountHT`, `tva`, `amountTTC`
- This pass runs before regex fallback, then existing regex remains as backup.

### 3) Field pattern cache self-heal
**File:** `src/main/java/com/invoice_reader/invoice_reader/servises/patterns/FieldPatternService.java`

- If cached pattern list is empty for a field, cache entry is cleared and reloaded.
- Added warning logs when no active patterns are loaded for a field.

### 4) SQL seed fixes for field patterns
**File:** `src/main/resources/data.sql`

- Fixed malformed `amountHT` insert separator that caused SQL init syntax error.
- Fixed two regex typos (`...\)?s*` -> `...\)?\s*`) in HT/TTC patterns.

## Expected Behavior After Fix
- `invoiceDate` should parse from OCR-noisy date lines and normalize correctly.
- Split totals blocks should map correctly:
  - `TotalHT` -> `amountHT`
  - `Total TVA ...` -> `tva`
  - `Total TTC ...` -> `amountTTC`
- Fallback from DB patterns is more resilient to temporary empty-cache states.
