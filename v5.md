# v5 - Session Auth + Roles + Dossier Access

## Summary
- Added session-based authentication (no password hashing, as requested).
- Added roles: `ADMIN`, `COMPTABLE`, `CLIENT`.
- Added `user_accounts` and `dossiers` tables, plus `dynamic_invoice.dossier_id`.
- Enforced role-based access and dossier ownership checks on invoice endpoints.
- Locked admin-only endpoints (accounts, tiers, templates, patterns, learning).

## New Data Model
- `user_accounts`
  - `username` (unique), `password` (plain text), `role`, `display_name`, `active`
- `dossiers`
  - `client_id` (FK -> user_accounts), `comptable_id` (FK -> user_accounts)
- `dynamic_invoice`
  - new column `dossier_id` (FK -> dossiers)

## Auth Endpoints
- `POST /api/auth/login`
  - body: `{ "username": "...", "password": "..." }`
  - sets session cookies
- `POST /api/auth/logout`
  - clears session
- `GET /api/auth/me`
  - returns current session user

## Seed Admin
- `data.sql` now inserts a default admin:
  - username: `admin`
  - password: `admin123`

## Frontend Session Usage
- Use `credentials: "include"` on login and all API calls so cookies are sent.

## Admin User Management
- `POST /api/auth/users` (ADMIN)
  - create user (any role)
- `GET /api/auth/users` (ADMIN)
  - list users
- `DELETE /api/auth/users/{id}` (ADMIN)
  - deactivate user

## Dossier Management (COMPTABLE / ADMIN)
- `POST /api/dossiers`
  - create client + dossier (or link existing client)
  - comptable uses own session
  - admin can specify `comptableId`
- `GET /api/dossiers`
  - list dossiers (admin = all, comptable = own)

## Invoice Access Rules
- **ADMIN**: full access to all invoices.
- **COMPTABLE**: only invoices where `dossier.comptable_id = session_user_id`.
- **CLIENT**: only invoices where `dossier.client_id = session_user_id`.

## Invoice Upload Notes
- `POST /api/dynamic-invoices/upload` and `/upload/batch`
  - optional `dossierId`
  - comptable/client must have a matching dossier (or a single dossier auto-selected).
  - admin may omit `dossierId` (invoice stays unassigned).

## Admin-Only Controllers
These now require `ADMIN` role:
- `AccountController`
- `TierController`
- `DynamicTemplateController`
- `FieldPatternController`
- `FieldLearningController`

## Files Changed / Added (high-level)
- `db.sql`: new tables + `dynamic_invoice.dossier_id`
- `DynamicInvoice` entity: dossier relation
- New auth/dossier entities, repositories, controllers, DTOs
- `SessionAuthInterceptor` + `@RequireRole` + `WebConfig`

## Security Note
Passwords are stored and compared in plain text (per request).  
This is **not safe for production** and should be replaced with hashing before deployment.
